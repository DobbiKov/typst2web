1. Introduction#

This document demonstrates the capabilities of typst-to-web, a tool that converts Typst documents into beautiful, self-contained HTML websites.

The approach uses the official Typst compiler to render pages as SVG — which means every Typst feature works perfectly: math, custom packages, cetz figures, and more.

1.1. Motivation#

Existing web compilers struggle with:

  • Complex mathematical expressions
  • Package-dependent content (cetz, fletcher, etc.)
  • Custom fonts and layouts

Our solution: delegate all rendering to the official Typst CLI.

5. Code Listings#

The tool is written in Python. The core compilation step:

import subprocess, tempfile
from pathlib import Path

def compile_to_svgs(typ_path):
    with tempfile.TemporaryDirectory() as tmp:
        out = Path(tmp) / "page-{p}.svg"
        subprocess.run(
            ["typst", "compile", str(typ_path), str(out), "--format", "svg"],
            check=True,
        )
        return [f.read_text() for f in sorted(Path(tmp).glob("page-*.svg"))]

6. Conclusion#

typst-to-web leverages the full power of the official Typst compiler, producing pixel-perfect SVG output that is then wrapped in a rich, interactive HTML viewer with:

  1. Full-text search across all pages
  2. Table of contents with click-to-navigate
  3. Dark / light mode toggle
  4. Zoom controls and keyboard navigation
  5. Print-friendly output
  6. Zero external runtime dependencies

All Typst features — including complex math ((eq. 2), (eq. 3)), cetz figures (Figure 1), and custom packages — are rendered correctly because we use the real compiler.
